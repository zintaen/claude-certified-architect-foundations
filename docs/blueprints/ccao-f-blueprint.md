## Learning objectives (outline, own words)

### Output Evaluation and Validation

- Spot hallucinations, hedging, and citation gaps in Claude outputs
- Decide when human review is required before acting on an answer
- Fact-check and bias-check outputs against source material

### Workflow Integration and Solution Design

- Choose when Claude Projects/knowledge sources suffice vs escalate to builders
- Shape workflows that balance quality, cost, and effort
- Scope Claude use-cases for non-engineering delivery roles

### Governance, Risk, and Responsible Use

- Apply acceptable-use and data-sensitivity judgement
- Identify high-stakes domains that need extra guardrails
- Escalate privacy or safety concerns appropriately

### Prompting and Task Execution

- Write clear task prompts for everyday professional work
- Iterate prompts when outputs miss the brief
- Use structured follow-ups without inventing facts

### Product and Model Selection

- Distinguish Claude.ai, Projects, Claude Code, and API surfaces at a product level
- Choose an appropriate model tier for a given task class
- Know when to hand off to a Developer or Architect

### Configuration and Knowledge Management

- Configure Projects with instructions and knowledge sources
- Keep knowledge bases current and scoped
- Avoid dumping sensitive data into prompts unnecessarily

### Troubleshooting and Optimisation

- Diagnose weak outputs (vague prompt, missing context, wrong surface)
- Reduce rework with clearer briefs and examples
- Know common failure modes of LLM assistants in office workflows
